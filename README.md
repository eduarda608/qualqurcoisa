O GitHub é a maior plataforma em nuvem do mundo para hospedagem de código-fonte e colaboração de projetos de software. Baseado no sistema Git, permite armazenar, compartilhar, revisar e gerenciar alterações em códigos de forma colaborativa. É essencial para controle de versão, portfólio profissional e desenvolvimento open source. 

O GitLab é uma plataforma DevSecOps completa baseada em Git, projetada para gerenciar todo o ciclo de vida de desenvolvimento de software em um único aplicativo. Ele permite colaboração em equipe, controle de versão, automação de CI/CD (integração e entrega contínuas), segurança e monitoramento de projetos. 

O Bitbucket é uma plataforma de hospedagem e gerenciamento de repositórios Git e Mercurial desenvolvida pela Atlassian, voltada para a colaboração em equipe. Ele permite armazenar código-fonte, realizar controle de versão, gerenciar projetos e implementar CI/CD (integração e implantação contínuas) com forte integração ao Jira e Trello. 

Compartilhamento no back-end refere-se ao compartilhamento de recursos, dados, lógica de negócios ou funcionalidades entre diferentes componentes, serviços, microserviços ou aplicações (frontend/mobile). Isso é realizado principalmente através de APIs RESTful, gerenciadores de pacotes ou banco de dados compartilhados. 

Hospedagem em back-end é o serviço de aluguel de servidores (na nuvem, VPS ou dedicados) onde o código de uma aplicação, bancos de dados e APIs são armazenados e executados para processar regras de negócio e dados. Diferente do front-end, foca nos bastidores (servidor) que processam solicitações, mantendo o sistema funcional e seguro. 

Ferramentas de back-end são tecnologias — linguagens, frameworks, bancos de dados e utilitários — usadas para construir a lógica, o gerenciamento de dados e a infraestrutura do lado do servidor de aplicações web e móveis. Elas processam solicitações, administram regras de negócios e conectam o front-end ao banco de dados, garantindo segurança e desempenho. 

A gestão em back-end é o gerenciamento da parte "invisível" de uma aplicação, focada nos bastidores (servidor, banco de dados e lógica de negócios) para garantir o funcionamento seguro, rápido e eficiente. Ela envolve criar APIs, gerenciar dados, autenticação de usuários e integrar sistemas, permitindo que a aplicação processe solicitações do usuário e armazene informações. 

Colaboração em back-end é o processo de trabalho em equipe para construir a lógica, bancos de dados e APIs que sustentam uma aplicação, garantindo que o servidor, o front-end e os sistemas integrados funcionem em sintonia. Envolve o uso de ferramentas como o Git no Atlassian para versionamento de código, comunicação constante com equipes de DevOps eフロント-end, e integração de regras de negócio de forma segura e eficiente. 

Um Pull Request (PR), ou "Solicitação de Puxada" em tradução livre, é um mecanismo no desenvolvimento de software — amplamente utilizado no back-end — onde um desenvolvedor notifica os membros da equipe que concluiu uma funcionalidade ou correção em uma ramificação (branch) separada.

ssue tracking (rastreamento de problemas) no desenvolvimento backend é o processo sistemático de registrar, monitorar e gerenciar erros, falhas, tarefas ou solicitações de recursos relacionados à lógica da aplicação, banco de dados, APIs ou infraestrutura do servidor. É uma parte crítica da manutenção de software, garantindo que bugs invisíveis ao usuário final (mas que quebram o sistema) sejam resolvidos de forma organizada.
